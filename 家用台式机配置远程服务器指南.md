# 家用台式机配置远程服务器指南（Tailscale 方案）

> 目标：用家里台式机跑 MySQL / MongoDB / Redis，通过 Tailscale 在任何地方免费访问。
> 无需公网 IP，无需路由器配置，5 分钟组网。

---

## 整体架构

```
家里台式机（服务端）
  ├── Docker 运行 MySQL / MongoDB / Redis / API
  └── Tailscale → 分配虚拟 IP 100.x.x.x

外出笔记本 / 手机（客户端）
  └── Tailscale 同账号登录 → 直接访问 100.x.x.x
```

---

## 第一步：台式机基础配置

### 1.1 设置静态内网 IP

防止重启后内网 IP 变化导致服务中断。

1. 右键任务栏网络图标 → 网络和 Internet 设置
2. 以太网 → 编辑 IP 分配 → 手动
3. 填入以下信息：

| 参数 | 值 |
|------|-----|
| IP 地址 | `192.168.1.100` |
| 子网掩码 | `255.255.255.0` |
| 网关 | `192.168.1.1` |
| DNS | `114.114.114.114` |

### 1.2 关闭睡眠和休眠

控制面板 → 电源选项 → 更改计划设置：
- 关闭显示器：**从不**
- 使计算机进入睡眠状态：**从不**

### 1.3 开放 Windows 防火墙端口

以**管理员身份**运行 CMD，依次执行：

```powershell
netsh advfirewall firewall add rule name="MySQL-3306" dir=in action=allow protocol=TCP localport=3306
netsh advfirewall firewall add rule name="MongoDB-27017" dir=in action=allow protocol=TCP localport=27017
netsh advfirewall firewall add rule name="Redis-6379" dir=in action=allow protocol=TCP localport=6379
netsh advfirewall firewall add rule name="FastAPI-8000" dir=in action=allow protocol=TCP localport=8000
```

## 第二步：安装 Tailscale

### 2.1 台式机安装

1. 下载：https://tailscale.com/download/windows
2. 安装后点击 Sign in，用 Google 或 GitHub 账号登录
3. 登录成功后，系统托盘出现 Tailscale 图标
4. 右键图标 → 查看虚拟 IP，格式为 `100.x.x.x`，**记下这个 IP**

### 2.2 其他设备安装

需要远程访问的设备（笔记本、手机等）都安装 Tailscale，登录**同一账号**：

| 平台 | 下载地址 |
|------|---------|
| Windows | https://tailscale.com/download/windows |
| macOS | https://tailscale.com/download/mac |
| Android | Google Play 搜索 Tailscale |
| iOS | App Store 搜索 Tailscale |

登录同一账号后，所有设备自动组网，可以直接用 `100.x.x.x` 互访。

### 2.3 设置 Tailscale 开机自启

右键系统托盘 Tailscale 图标 → Preferences → 勾选 **Run at startup**

## 第三步：部署项目服务

### 3.1 安装 Docker Desktop

1. 下载：https://www.docker.com/products/docker-desktop/
2. 安装完成后重启电脑
3. 启动 Docker Desktop，确认右下角图标变绿
4. Settings → General → 勾选 **Start Docker Desktop when you sign in**

### 3.2 克隆项目

```bash
git clone https://gitee.com/penney-101/ai-novel-matrix-studio.git
cd ai-novel-matrix-studio\ai_novel_studio
```

### 3.3 配置环境变量

```bash
copy ..\config.env.example .env
notepad .env
```

填入以下内容（容器内部通信用服务名，不用 IP）：

```env
MYSQL_ROOT_PASSWORD=设置一个强密码
MYSQL_URL=mysql+aiomysql://root:强密码@mysql:3306/ai_novel_studio
MONGODB_URL=mongodb://mongodb:27017
REDIS_URL=redis://redis:6379/0

MINIMAX_API_KEY=你的key
DOUBAO_API_KEY=你的key
QWEN_API_KEY=你的key
DEEPSEEK_API_KEY=你的key
```

### 3.4 启动所有服务

```bash
docker compose up -d

# 查看运行状态，所有服务应显示 healthy 或 running
docker compose ps
```

### 3.5 配置 MySQL 允许远程连接

```bash
# 进入 MySQL 容器
docker exec -it ai_novel_studio-mysql-1 mysql -uroot -p

# 输入密码后执行：
GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' WITH GRANT OPTION;
FLUSH PRIVILEGES;
EXIT;
```

## 第四步：配置开机自动启动服务

新建文件 `启动服务.bat`，内容如下：

```bat
@echo off
echo 正在启动 AI小说工作室服务...
cd /d D:\codeSpace\ai-novel-matrix-studio\ai_novel_studio
docker compose up -d
echo 服务启动完成！
```

将此文件的**快捷方式**放入开机启动目录：

```
C:\Users\你的用户名\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup
```

---

## 第五步：更新 config.env 使用远程地址

在**外出设备**（笔记本等）上，把 `config.env` 改成台式机的 Tailscale 虚拟 IP：

```env
DB_HOST=100.x.x.x
DB_PORT=3306
DB_USER=root
DB_PASS=你设置的强密码
DB_NAME=ai_novel_studio

MYSQL_URL=mysql+aiomysql://root:强密码@100.x.x.x:3306/ai_novel_studio
MYSQL_SYNC_URL=mysql+pymysql://root:强密码@100.x.x.x:3306/ai_novel_studio?charset=utf8mb4
MONGODB_URL=mongodb://100.x.x.x:27017
REDIS_URL=redis://100.x.x.x:6379/0
```

> `100.x.x.x` 替换为第二步记下的 Tailscale 虚拟 IP

## 第六步：验证连接

### 测试 MySQL

```bash
mysql -h 100.x.x.x -P 3306 -u root -p
```

### 测试 API

浏览器访问：
```
http://100.x.x.x:8000/health
```

返回 `{"status": "ok"}` 说明正常。

### 测试 GUI

```bash
python -m ai_novel_studio.gui.app
```

进入系统设置页面，把数据库地址改为 `100.x.x.x`，点击测试连接。

## 安全建议

| 项目 | 建议 |
|------|------|
| MySQL 密码 | 不要用 `root`，设置 12 位以上含大小写+数字+符号的强密码 |
| Redis 密码 | docker-compose.yml 的 redis 服务加：`command: redis-server --requirepass 你的密码` |
| MongoDB | 只通过 Tailscale 访问，不要对外暴露 27017 端口 |
| 定期备份 | 每周手动导出一次 MySQL：`docker exec mysql mysqldump -uroot -p ai_novel_studio > backup.sql` |

## 常见问题

**Q: Tailscale 连上了但数据库连不上？**
- 检查 Windows 防火墙是否开放了 3306 端口（第一步 1.3）
- 确认 Docker 容器正在运行：`docker compose ps`
- 确认执行了 MySQL 远程授权（第三步 3.5）

**Q: 台式机重启后服务没了？**
- 确认 Docker Desktop 已设置开机自启
- 确认 `启动服务.bat` 快捷方式已放入启动目录

**Q: Tailscale 虚拟 IP 会变吗？**
- 不会，同一设备的 Tailscale IP 是固定的，不会因重启或断网而改变

**Q: 台式机关机了怎么办？**
- 台式机必须保持开机状态才能访问服务
- 可以在路由器设置 WOL（网络唤醒），用手机远程开机

## 备选方案：阿里云 RDS MySQL

> 适合场景：不想让台式机长期开机，或需要更稳定的数据库服务。
> 费用：约 ¥70-100/月（按量付费约 ¥0.2/小时）

### 方案对比

| 对比项 | 家里台式机 | 阿里云 RDS |
|--------|-----------|-----------|
| 费用 | 电费 ¥30-80/月 | ¥70-100/月 |
| 稳定性 | 依赖台式机开机 | 24小时稳定运行 |
| 维护 | 需自己管理 | 全托管，零维护 |
| 备份 | 手动备份 | 自动每日备份 |
| 延迟 | Tailscale 虚拟网络 | 直连公网，延迟低 |
| 适合 | 开发测试 / 个人使用 | 正式上线 / 团队使用 |

### RDS 配置步骤

#### 第一步：创建 RDS 实例

1. 登录阿里云控制台：https://console.aliyun.com
2. 搜索 **RDS** → 云数据库 RDS → 创建实例
3. 选择配置：

| 参数 | 推荐值 |
|------|--------|
| 数据库类型 | MySQL 8.0 |
| 系列 | 基础系列（单节点，够用）|
| 规格 | 1核2G（最低配，约 ¥70/月）|
| 存储 | 20G SSD |
| 地域 | 选离你最近的（如华东-上海）|
| 计费方式 | 按量付费（灵活，随时释放）|

4. 点击立即购买，等待实例创建完成（约 3-5 分钟）

#### 第二步：创建数据库和账号

进入 RDS 实例控制台：

1. 左侧菜单 → **数据库管理** → 创建数据库
   - 数据库名：`ai_novel_studio`
   - 字符集：`utf8mb4`

2. 左侧菜单 → **账号管理** → 创建账号
   - 账号名：`novel_user`
   - 账号类型：普通账号
   - 密码：设置强密码
   - 授权数据库：选择 `ai_novel_studio`，权限选**读写**

#### 第三步：配置白名单

左侧菜单 → **数据安全性** → 白名单设置 → 添加白名单分组：

- 开发阶段：填入你的本地公网 IP（访问 https://ifconfig.me 查看）
- 或填入 `0.0.0.0/0` 允许所有 IP（不推荐生产环境）

#### 第四步：获取连接地址

左侧菜单 → **基本信息** → 连接信息：

- **内网地址**：只能在同地域阿里云服务器访问
- **外网地址**：任何地方都能访问（需手动申请，点击"申请外网地址"）

记下外网地址，格式类似：
```
rm-xxxxxx.mysql.rds.aliyuncs.com
```

#### 第五步：导入数据库结构

```bash
# 本地执行，把建表脚本导入 RDS
mysql -h rm-xxxxxx.mysql.rds.aliyuncs.com -P 3306 -u novel_user -p ai_novel_studio < ai_novel_studio/database/schema.sql
```

#### 第六步：更新 config.env

```env
DB_HOST=rm-xxxxxx.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_USER=novel_user
DB_PASS=你设置的强密码
DB_NAME=ai_novel_studio

MYSQL_URL=mysql+aiomysql://novel_user:强密码@rm-xxxxxx.mysql.rds.aliyuncs.com:3306/ai_novel_studio
MYSQL_SYNC_URL=mysql+pymysql://novel_user:强密码@rm-xxxxxx.mysql.rds.aliyuncs.com:3306/ai_novel_studio?charset=utf8mb4
```

> MongoDB 和 Redis 仍可继续跑在台式机上，通过 Tailscale 访问，只把 MySQL 迁移到 RDS。

#### 第七步：验证连接

```bash
mysql -h rm-xxxxxx.mysql.rds.aliyuncs.com -P 3306 -u novel_user -p
```

连接成功后启动 GUI 测试：

```bash
python -m ai_novel_studio.gui.app
```

进入系统设置，填入 RDS 连接信息，点击测试连接。

### 费用控制建议

- **按量付费**：不用时可以暂停实例，停止计费（数据保留）
- **包年包月**：长期使用可切换包月，约 ¥70/月，比按量便宜 30%
- **存储单独计费**：20G 约 ¥3/月，够用就不要扩容
- **不需要外网时**：关闭外网地址，只保留内网访问，更安全也省钱
