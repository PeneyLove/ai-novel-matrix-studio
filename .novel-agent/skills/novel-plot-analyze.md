# 剧情分析诊断

## 角色定位
对已有章节进行全面剧情健康度检查，输出量化诊断报告。

## 操作步骤
1. `read_file .novel-agent/state.json` 获取总章节数
2. `read_file outlines/main_outline.txt` 获取大纲基准
3. `read_file .novel-agent/hooks/ledger.yaml` 获取伏笔台账
4. 逐章或抽样读取（`read_file chapters/第N章/chapter.txt`）
5. 输出诊断报告，用户确认后按建议逐一修复

## 诊断维度

### 一、结构完整性（满分10）
- 开篇钩子（第1章前500字是否吸引人）
- 每卷起承转合是否完整
- 高潮分布是否均匀（每20-30章一大高潮）
- 结局设定是否存在（即使还没写到，大纲里有没有）

### 二、伏笔健康度（满分10）
- 已埋伏笔总数 / 已回收数 / 回收率
- 逾期未回收数（超过预定章的）
- 悬空伏笔（回收计划为空或无明确回收章）
- 新埋伏笔速度（最近20章新增伏笔数）

### 三、节奏健康度（满分10）
- 近20章平均每章爽点/冲突数
- 连3章以上无高潮的段落
- 连2章以上连续高潮的段落（可能过赶）
- 每章结尾钩子覆盖率

### 四、人物一致度（满分10）
- 主要角色是否有OOC行为
- 配角出场频率（是否有的角色"消失"超过20章）
- 人物关系进展（近50章内的关系变化）

## 输出模板
```
═══ 剧情健康度诊断 ═══
总分：[X/40]

【结构完整性】[X/10]
· [具体发现1]
· [具体发现2]

【伏笔健康度】[X/10]
· 埋数/收数/回收率
· 逾期：[N条]
· 悬空：[N条]

【节奏健康度】[X/10]
· 平均爽点密度
· 问题段落：[位置]

【人物一致度】[X/10]
· OOC风险：[角色名]
· 消失角色：[角色名]

═══ 优先修复建议 ═══
1. [最高优先级]
2. [次优先级]
```
