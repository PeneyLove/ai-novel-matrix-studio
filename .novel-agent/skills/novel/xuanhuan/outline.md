# 玄幻修仙 — 大纲全链路搭建 & 迭代

## 角色定位
基于已确认的类型设定，生成完整大纲。大纲未定稿前绝对不启动正文写作。

## 前置条件
- 读取 `.novel-agent/state.json`，确认 genre=xuanhuan 且 phase=init
- 如 phase 不是 init，提示用户先执行 `/xuanhuan-genre_init`

## 操作步骤

1. **读取状态**：`read_file .novel-agent/state.json`
2. **生成大纲**：按下方模板输出完整大纲
3. **等待审核**：用户提出修改意见后迭代更新，每轮标注变更
4. **定稿确认**：用户说"定稿"或"可以了"后：
   - 用 `write_file` 写入 `outlines/main_outline.txt`（UTF-8, Markdown）
   - 用 `write_file` 更新 `.novel-agent/state.json`，phase 改为 outline
5. **引导下一步**：告知用户执行 `/xuanhuan-hooks`

## 玄幻修仙大纲规范

### 必须覆盖的模块
- **核心设定**：境界体系（完整链条+每级特点）、力量体系（灵根/功法/丹药/法宝/阵法）、世界地图（至少3个区域）
- **人物谱系**：主角（全名+身世+金手指+性格+成长弧线）+ ≥4配角（每人定位+与主角关系+性格标签）+ ≥3反派（每人动机+势力+压迫感）
- **主线剧情**：核心冲突→分卷剧情（5-8卷）→终极结局
- **爽点节点**：每卷标记3个核心爽点（类型：升级/打脸/逆袭/夺宝/突破/揭秘/扬名）
- **感情线**：≥1条（可选纯剧情流无感情线）

### 迭代规则
- 每次修改输出完整更新后的大纲
- 标注本轮变更位置（用 `【变更】` 标记）
- 同步更新关联设定（改人设→影响剧情节点）
- 用户说定稿后锁定版本号

## 输出模板

```
当前调用 Skill：玄幻修仙-大纲全链路搭建 & 迭代 Skill

═══ 《小说名》完整大纲 ═══

【核心设定】
· 境界体系：[详细链条+每级说明]
· 力量体系：[灵根/功法/丹药/法宝/阵法/秘境]
· 世界观：[至少2界/域，每界3-5个关键地点]
· 独特设定亮点：[区别于同类型的创新点]

【人物谱系】
■ 主角：[全名] [身世] [金手指] [性格] [成长弧线]
■ 配角1-4：[姓名/定位/关系/性格]
■ 反派1-3：[姓名/动机/势力/压迫感]

【主线剧情】
· 核心冲突：[一句话概括]
· 第一卷：[卷名]（第1-40章）[核心事件+结局]
· 第二卷：[卷名]（第41-80章）...
· ...（5-8卷）
· 终极结局：[主角最终状态+世界变化+感情线结局]

【爽点节点】
· 卷一爽点：①[类型+章节] ②[类型+章节] ③[类型+章节]
· ...

═══ 大纲状态：待审核 ═══
请逐项审核，提出修改意见。确认无修改后说"定稿"即可锁定。
```
