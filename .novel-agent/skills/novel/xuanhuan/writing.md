# 玄幻修仙 — 正文定向续写（大纲绑定版）

## 角色定位
基于已定稿的大纲和钩子台账，逐章续写玄幻修仙小说正文。严格绑定大纲，绝不跑偏。

## 前置条件
- `outlines/main_outline.txt` — 已定稿大纲
- `.novel-agent/hooks/ledger.yaml` — 钩子台账
- `.novel-agent/state.json` — 当前进度

## 操作步骤

1. **读取进度**：`read_file .novel-agent/state.json` → 获取 chapter 和 total_chapters
2. **读取大纲**：`read_file outlines/main_outline.txt` → 获取当前卷的剧情纲要
3. **读取台账**：`read_file .novel-agent/hooks/ledger.yaml` → 检查是否需要回收/埋伏笔
4. **读取上文**：如有上一章，读取最后500字做上下文衔接
5. **撰写章节**：按规范创作，输出到对话中供用户审核
6. **用户确认后持久化**：
   - `bash mkdir -p "chapters/第N章/"`
   - `write_file chapters/第N章/chapter.txt` → 写入正文章节（UTF-8, Markdown）
   - `write_file .novel-agent/state.json` → 更新 chapter（chapter号+1）
   - 如有新埋伏笔或回收，`write_file .novel-agent/hooks/ledger.yaml`

## 玄幻修仙续写规范

### 章节结构（每章2000-4000字）
1. **钩子回收**（100-200字）：自然地衔接上一章的收尾钩子
2. **本章正文**（1500-3500字）：推进剧情/战斗/修炼/对话
3. **微爽点植入**：每章≥1处，类型轮换（升级突破/打脸逆袭/获得机缘/揭秘真相/实力展示/扬名立万）
4. **伏笔铺垫**：每章1处新的小伏笔/线索
5. **收尾钩子**（50-100字）：悬念/冲突/揭秘/危机/奖励型钩子

### 写作风格
- **对话快节奏**：对话占30-40%，每轮对话不超过3句
- **战斗有层次**：每次战斗≥3个回合，描写具体招式而非"一拳轰出"
- **修炼有细节**：突破时描写体内变化、天地异象、瓶颈感
- **禁止AI套话**：不许"不仅如此""更重要的是""总而言之"等
- **人物有辨识度**：通过口头禅、动作习惯、思维方式区分角色
- **网文节奏**：每500字内必有一个小冲突/爽点/悬念
- **环境描写**：每3000字至少1次场景切换+环境渲染

### 常用元素模板
- **突破描写**：灵力暴涌→经脉承受→瓶颈松动→天地共鸣→突破成功+新能力展示
- **战斗描写**：试探交锋→被压制→激发潜力/使用底牌→逆转→代价/收获
- **打脸描写**：被轻视→挑衅升级→主角展示实力→众人震惊→后续影响
- **机缘获取**：发现线索→探索险境→破解机关→获得宝物→遭遇守护者/追兵
