# 玄幻修仙 — 伏笔/爽点/钩子全维度埋置

## 角色定位
基于已定稿的大纲，系统性预埋伏笔、设计爽点节奏、预留章节钩子，建立可追踪的钩子台账。

## 前置条件
- `outlines/main_outline.txt` 必须存在且已定稿
- `.novel-agent/state.json` phase=outline

## 操作步骤

1. **读取大纲**：`read_file outlines/main_outline.txt`
2. **读取状态**：`read_file .novel-agent/state.json`
3. **生成钩子台账**：按规范输出三类埋点
4. **用户确认后持久化**：
   - 用 `write_file` 写入 `.novel-agent/hooks/ledger.yaml`（UTF-8 YAML）
   - 用 `write_file` 更新 `.novel-agent/state.json`，phase 改为 hooks
5. **引导下一步**：告知用户可执行 `/novel-continue` 开始正文

## 玄幻修仙钩子规范

### 一、长线伏笔（≥10条）
- 编号：hook-001 ~ hook-010+
- 每条必须标注：ID / 内容描述 / 埋伏章节 / 计划回收章节 / 反转程度（强/中/弱）
- 分类：身份谜团（≥2条）/ 上古秘辛（≥2条）/ 敌人暗线（≥2条）/ 功法因果（≥2条）/ 感情暗线（≥2条）

### 二、阶段性爽点布局
- 每卷3个核心爽点 → 标注类型和预期效果
- 每5章至少1个小爽点 → 交替使用：升级/打脸/夺宝/突破/揭秘/扬名/救人/报仇
- 爽点密度曲线：每卷呈上升趋势

### 三、章节钩子模板
每章末尾预留以下类型钩子（轮换使用）：
- 悬念型：揭露未知信息的一角
- 冲突型：矛盾升级/新敌出现
- 揭秘型：揭示一个小真相
- 危机型：主角陷入困境
- 奖励型：暗示即将获得的机缘
