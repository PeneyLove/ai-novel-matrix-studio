# 科幻无限 — 爽点强化

## 操作步骤
1. `read_file` 读取目标章节
2. 按诊断标准分析 → 输出诊断报告
3. 用户确认后 `write_file` 覆写优化后内容

## 诊断标准
- **密度检查**：每章至少3处情绪峰值（爽点/燃点/突破感）
- **强度检查**：核心爽点渲染力度（前置铺垫≥200字/高潮描写≥300字/余韵≥100字）
- **类型配比**：避免全是同类型爽点，轮换使用：升级突破/越级杀敌/机缘获得/打脸逆袭/扬名立万
- **水字数识别**：超过300字无剧情推进/无情绪波动的段落标记为水字数

## 修复策略
- 薄弱爽点：增加铺垫细节→拉长高潮描写→加入围观者反应→增加后续影响
- 水字数：压缩为1句过渡→补齐微爽点
- 具体元素：副本通关/科技对决/星际战争/末世求生/系统升级

## 输出模板
```
当前调用 Skill：科幻无限-爽点强化

═══ 爽点诊断 ═══
· 爽点密度：[X处/章]
· 最强爽点：[位置+类型+渲染力度评分]
· 最弱爽点：[位置+原因]
· 水字数段落：[位置+字数]

═══ 优化后内容 ═══
[完整优化文本]
```
