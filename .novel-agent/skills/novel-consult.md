# 内置创作咨询引擎

## 角色定位
使用内置结构化多源分析引擎，对已完成的创作内容进行确定性检查，输出量化评分和改进建议。与依赖模型直觉的分析不同，本咨询基于预定义规则和模式匹配，结果一致、可重复，且支持同时分析多个源文件（大纲 + 人设 + 伏笔台账 + 最新章节）的综合诊断。

## 可用分析维度
| 维度 | 策略名 | 说明 |
|------|--------|------|
| 🏗 大纲完整性 | `outline-completeness` | 检查【核心设定/人物谱系/主线剧情/爽点节点】等模块完整性 |
| 👤 人设一致性 | `character-consistency` | 检查人设YAML与正文中角色行为是否一致，标记OOC风险 |
| 📖 剧情结构 | `plot-structure` | 检查核心冲突、分卷规划、结局完整性 |
| 🎵 节奏健康 | `pacing-health` | 检查爽点密度、高潮分布、钩子覆盖率 |
| 🔍 逻辑排查 | `logic-debug` | 检查时间线矛盾、战力崩坏、设定冲突 |
| 🪝 伏笔回收 | `hook-recovery` | 检查伏笔台账的回收率、逾期伏笔、悬空伏笔 |
| ✍️ 风格分析 | `style-analysis` | 检查AI-slop短语、段落长度、叙述视角 |

## 使用方式
`/novel-consult {target}` — 支持的 target：

| target | 读取文件 | 分析维度 |
|--------|----------|----------|
| `outline` 或 `大纲` | `outlines/main_outline.txt` | 大纲完整性 + 剧情结构 + 节奏健康 |
| `characters` 或 `人设` | `characters/*.yaml` + 最近5章 | 人设一致性 |
| `plot` 或 `剧情` | `outlines/main_outline.txt` + 最近章节 | 剧情结构 + 节奏健康 + 逻辑排查 |
| `hooks` 或 `伏笔` | `.novel-agent/hooks/ledger.yaml` | 伏笔回收校验 |
| `style` 或 `风格` | 最近5章 | 风格分析 |
| `full` 或 `完整`或 `all` | 上述全部文件 | 全部维度 |

## 执行步骤

### 1. 读取材料
- `read_file .novel-agent/state.json` 获取当前状态
- 根据 target 读取对应文件（见上表）
- 对于 `full` 模式，依次读取所有需要的文件

### 2. 调用内置分析引擎
使用 `novel_consult` 工具传入分析内容。对于多源分析（`full`/`all`），将不同源的内容用明确的分隔标记拼接后传入。

**单源分析调用示例（大纲审核）：**
```
novel_consult(
  subject="大纲完整性检查",
  source="<outline_file_content>"
)
```

**多源分析调用示例（完整检查）：**
将多个文件内容分段拼接：
```
novel_consult(
  subject="完整创作咨询",
  source="=== outline ===\n<大纲内容>\n\n=== characters ===\n<人设内容>\n\n=== hooks ===\n<伏笔台账>\n\n=== chapters ===\n<章节内容>"
)
```

分析引擎会自动识别各源并运行对应策略。

### 3. 输出咨询报告
按下方模板输出结构化报告。

## 输出模板
```
═══ 创作咨询报告 ═══
分析对象：{target}
健康评分：{score}/100
总体评估：{summary}

共发现 {N} 个问题：

### 🔴 必须修复（阻塞项）
{逐条列出}

### 🟡 建议修复（警告）
{逐条列出}

### 🔵 参考建议（信息）
{逐条列出}

═══ 报告结束 ═══
```

## 每条问题格式
```
**{标题}**（{位置}）
> {详细描述}
💡 建议：{具体修复建议}
可信度：{█████ 90%}
```

## 注意事项
- 分析基于确定性规则，不依赖模型猜测
- 多条分析比单条分析更全面但可能包含冗余信息，full 模式下会综合所有维度
- 如果用户对某条建议有疑问，协助解释判断依据
- 用户采纳建议后，协助实际修改对应文件
- 咨询结果为参考性质，最终决策权在用户
