# 卡文推演 Skill

## 角色定位
当剧情推进卡顿或面临分支选择时，发挥长链推理优势，多路径推演并推荐最优方案。

## 前置条件
- `read_file .novel-agent/state.json` 确认当前阶段和章节
- `read_file outlines/main_outline.txt` 了解大纲框架
- `read_file .novel-agent/hooks/ledger.yaml` 了解待回收伏笔

可选读取最近章节：
- 使用 `ls chapters/` 获取最新章节目录
- `read_file chapters/第N章/chapter.txt` 读取前几章内容

## 操作步骤

### 1. 分析当前剧情节点
- 当前主要冲突是什么
- 主角当前状态（境界/能力/关系）
- 有哪些未回收的伏笔可以激活
- 读者当前的情绪预期（爽点满足度）

### 2. 生成 3 条可行推进路径
每条路径包含：
- **路径名**：一句话概括
- **核心事件**：接下来 3-5 章的关键事件链
- **爽点类型**：该路径的主打爽点
- **关键转折**：该路径的核心反转点

### 3. 多维度评估每条路径

| 维度 | 说明 | 评分标准 1-10 |
|------|------|---------------|
| 爽点强度 | 情绪的爆发力度 | 10=拍案叫绝 |
| 逻辑合理性 | 是否符合已有设定 | 10=严丝合缝 |
| 后续延展性 | 能否引出更多剧情 | 10=可发展20章+ |
| 人设契合度 | 是否符合角色性格 | 10=角色必然如此 |
| 新颖度 | 是否落入俗套 | 10=意想不到 |

### 4. 输出推演报告

```
═══ 卡文推演报告 ═══
当前节点：第{N}章 — {当前剧情摘要}

━━━ 路径 A：{路径名} ━━━
核心事件链：
1. 第{N+1}章：{事件}
2. 第{N+2}章：{事件}
3. 第{N+3}章：{事件}

爽点类型：{类型}
关键转折：{描述}

评分：爽点{ X/10} · 逻辑{ X/10} · 延展{ X/10} · 人设{ X/10} · 新颖{ X/10}
综合推荐指数：{X/10}

━━━ 路径 B：{路径名} ━━━
...

━━━ 路径 C：{路径名} ━━━
...

═══ 推荐方案 ═══
首选路径：路径 {X}
理由：{详细理由}
```

## 输出
用户确认后，可选直接生成后续 3 章细纲（调用 `/xuanhuan-outline` 类的细纲技能）
