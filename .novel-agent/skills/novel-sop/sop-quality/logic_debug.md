# 逻辑 bug 排查 Skill

## 角色定位
使用内置确定性分析引擎检查剧情逻辑问题：时间线矛盾、战力体系崩坏、设定前后冲突。与单纯依赖模型感觉的判断不同，本检查基于结构化规则和模式匹配。

## 前置条件
- `read_file .novel-agent/state.json`
- `read_file outlines/main_outline.txt`
- 读取最近 10 章内容

## 检查维度

### 1. 时间线一致性
- 事件顺序是否符合因果关系
- 是否有同一时间发生两件矛盾的事
- 时间跳跃是否有说明

### 2. 战力体系
- 同一境界/等级内的战力是否合理
- 主角升级速度是否过快或过慢
- 是否出现「刚突破又突破」的节奏问题

### 3. 设定一致性
- 已有设定是否被后续章节推翻
- 角色能力是否按设定逻辑使用

## 操作步骤

### 1. 读取材料
```
read_file outlines/main_outline.txt
```
读取最近 5-10 章内容。

### 2. 调用内置分析
使用 `novel_consult` 工具：
```
novel_consult(
  subject="逻辑排查",
  source="<章节内容>"
)
```

### 3. 输出排查结果
```
═══ 逻辑排查报告 ═══
检查范围：第{X}-{Y}章

【时间线】
✓ 事件顺序合理
⚠ 第{N}章和第{N+1}章之间的时间跳跃未说明

【战力体系】
✓ 升级节奏正常
⚠ 「{境界}」突破次数偏多（{N}次）

【设定一致性】
✓ 未发现冲突
```
