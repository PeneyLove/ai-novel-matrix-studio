# 对标作品拆解 Skill

## 角色定位
按网文 SOP 标准拆解对标作品，输出结构化赛道数据，为选题和卖点设计提供依据。

## 输入
- `target_platform`: 目标平台（起点/七猫/番茄/晋江/飞卢）
- `category`: 题材分类（玄幻/都市/古言/悬疑/科幻/甜宠）
- `reference_work`: 对标作品名（用户提供名称，非文本）
- `review_method`: 分析方式 — `read_file` 读取已有拆书文本，或用户手动描述

## 核心逻辑

### 1. 获取输入
与用户对话确认：
- 目标平台和目标题材
- 对标作品名称和核心亮点（用户知道即可，不需要全文）
- 用户已知的该作品关键数据（首日收藏、推荐票趋势等）

### 2. 标准化拆解维度
按以下维度结构化分析（基于用户描述）：

**开篇分析：**
- 首章冲突入场方式（直接冲突/世界观铺垫/悬念引入）
- 金手指出场时间（第几章、什么形式）
- 第1个小高潮位置（第几章、什么事件）

**节奏分析：**
- 爽点密度估算（平均每几章一个核心爽点）
- 爽点类型分布（升级/打脸/逆袭/夺宝/突破/揭秘/扬名）
- 每章结尾钩子设计方式

**人设分析：**
- 主角人设模型（身份+性格+金手指三者关系）
- 配角工具人指数（是否有独立弧光）
- 反派压迫感设计

### 3. 输出结构化报告

```
═══ 对标作品拆解报告 ═══
作品名：{作品名}
题材：{题材}
平台：{平台}

【开篇拆解】
· 首章冲突：{描述}
· 金手指：{类型+出场章}
· 第1个小高潮：{章+事件}

【节奏特征】
· 爽点密度：约每{X}章一个
· 主要爽点类型：{类型列表}
· 章末钩子：{设计模式}

【人设分析】
· 主角模型：{身份+性格+金手指}
· 配角功能：{工具人指数}
· 反派设计：{压迫感类型}

【可复用桥段】
· {桥段1}
· {桥段2}

═══ 报告结束 ═══
```

## 输出
- 结构化拆书报告（写入 `outlines/benchmark/{work_name}.md`）
- 可复用桥段标签列表
