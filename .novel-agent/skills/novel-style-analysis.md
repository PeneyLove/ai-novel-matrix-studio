# 文风分析与优化

## 角色定位
分析作者已有章节的文风特征，帮助建立统一风格，识别并修正语言问题。

## 操作步骤
1. `read_file` 读取目标章节（建议≥3章以获得准确文风画像）
2. 逐项分析 → 输出诊断报告
3. 用户确认后 `write_file` 应用语言优化

## 分析维度

### 一、叙事视角
- **人称**：第一/第三（限知/全知）
- **视角切换**：是否在同一章内频繁切换视角
- **内心独白占比**：过于依赖内心独白会导致"告诉"而非"展示"

### 二、对话风格
- **对话占比**：<30%对话偏少，>60%对话过多
- **辨识度**：不同角色的对话是否可区分（不看标注能认出谁在说话？）
- **对话标签**：是否过度使用"他说""她说" — 建议用动作替代
- **信息密度**：对话中是否夹带过多说明性文字（"如你所知..."是典型问题）

### 三、描写风格
- **白描 vs 浓墨**：环境描写的篇幅占比
- **感官运用**：视觉/听觉/嗅觉/触觉/味觉的分布
- **节奏感**：长短句交替情况，是否有过多长句导致阅读疲劳
- **抽象 vs 具体**：是否用具体场景代替抽象概括

### 四、语言问题检测
- **AI套话**：以下是常见 AI 写作痕迹，一旦发现必须删除：
  - "不仅如此" "更重要的是" "总而言之" "综上所述"
  - "在这个过程中" "从此以后" "毫无疑问"
  - 结尾总结句（如"这次经历让他明白..."）
- **重复用词**：同一段内重复≥3次的词汇
- **被动语态**：过多"被"字句 → 改为主动
- **成语堆砌**：连用≥2个成语 → 保留1个，其余用描写替代

### 五、网文特有检查
- **章节钩子**：结尾是否有悬念/冲突/揭秘/危机
- **分段长度**：网文每段≤5行（手机阅读友好）
- **开篇吸引力**：前500字是否抛出钩子
- **章节标题**：是否有吸引力（可选）

## 输出模板
```
═══ 文风诊断 ═══

【叙事视角】[视角类型/切换频率/内心独白占比]
【对话风格】[占比/辨识度/标签使用/信息密度]
【描写风格】[白描vs浓墨/感官分布/节奏/具体性]

【语言问题】
· AI套话：[N处，列出位置]
· 重复用词：[词汇+次数]
· 被动语态：[N处]
· 成语堆砌：[N处]

【网文优化】
· 章节钩子：[强度评级]
· 分段：[是否手机友好]
· 开篇：[吸引力评级]

═══ 优化后文风统一为 ═══
[统一后的文风特征描述]
```
